--------------------------------------------------------------------
1. 빌드 도구 설치 (블로커 해제)
--------------------------------------------------------------------
초기 상태: cmake/gcc/clang/SDL2 전부 미설치 (check_build.ps1 → exit 2)

설치 절차 (Windows / PowerShell):
  1) winget install --id MSYS2.MSYS2   → MSYS2 20260611 설치 성공
  2) C:\msys64\usr\bin\bash.exe -lc "pacman -Sy --noconfirm --needed \
        mingw-w64-x86_64-toolchain mingw-w64-x86_64-cmake mingw-w64-x86_64-SDL2"
     → gcc 16.1.0, cmake 4.3.4, SDL2 2.32.10 설치 성공
  3) 세션 PATH 에 C:\msys64\mingw64\bin 추가 → cmake/gcc/sdl2-config 인식 확인

--------------------------------------------------------------------
2. 빌드 검증 결과 (실제 수행)
--------------------------------------------------------------------
$ cmake -S src/simulator -B build_sim -G "MinGW Makefiles" -DCMAKE_PREFIX_PATH="C:/msys64/mingw64"
  → Configuring done / Generating done (SDL2 발견, LVGL release/v9.2 FetchContent 다운로드)
$ cmake --build build_sim
  → [100%] Built target lvgl / lvgl_simulator / lvgl_thorvg  (BUILD_EXIT=0)

$ src/simulator/check_build.ps1
  → PASS: lvgl_simulator 실행 파일 생성 (exit 0)

산출 실행파일: build_sim/bin/lvgl_simulator.exe (약 1.20 MB)

빌드 중 해결한 이슈 (3라운드):
  R1) LVGL custom.cmake 가 ARM 전용 어셈블리(*.S: helium/neon)를 무조건 glob →
      x86 MinGW 어셈블러 실패. 대처: CMakeLists.txt 에서 lvgl 타깃의 .S 소스 제거
      + 예제/데모 비활성화(LV_CONF_BUILD_DISABLE_EXAMPLES/DEMOS).
  R2) main.c 의 #include "lvgl/lvgl.h" → LVGL include 루트 기준 "lvgl.h" 로 수정.
  R3) SDL2main 이 SDL_main 심볼 요구(undefined reference). 대처: main.c 에
      SDL_MAIN_HANDLED + SDL_SetMainReady(), CMake 는 SDL2::SDL2(core) 링크.

--------------------------------------------------------------------
3. 실행/창 팝업 검증
--------------------------------------------------------------------
$ Start-Process build_sim\bin\lvgl_simulator.exe (4초 구동 후 종료)
  → 프로세스 정상 기동, stderr 없음(= SDL 윈도우 생성 성공; 실패 시 main.c 가
    "SDL2 윈도우 생성 실패" 출력 후 종료함).
  → 실행 중 스크린샷 저장: docs/verification/T-801_pc_simulator_run.png

한계 명시: 이 실행은 자동화 세션에서 수행되어 창이 실제 데스크톱 전면에 표시되는지
  육안 확인은 제한적이다. 다만 (a) 빌드/링크 성공, (b) 실행파일 생성, (c) 프로세스가
  4초간 SDL 에러 없이 구동됨 → 스캐폴딩 검증 목적(카드 10항 핵심)은 달성으로 판단.
