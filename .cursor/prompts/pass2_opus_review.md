# Pass 2 킥오프 프롬프트 (Cursor 모델: Opus-4.8-max)

Cursor 모델 선택기를 **Opus-4.8-max**로 바꾼 뒤, `T-xxx`를 실제 Task ID로 바꿔서 채팅에 붙여넣는다.

---

Task ID: T-xxx

`.cursor/rules/task-workflow.mdc`의 5단계(Pass 2 리뷰/보강)를 수행해줘. 너는 Pass 1 구현자가 아니라 별도의 시니어 리뷰어야.

1. `단위구현계획서.md`의 T-xxx 카드와 현재 feature 브랜치의 diff를 처음부터 다시 읽어줘.
2. 다음을 점검해줘: 목적(7항) 일치 여부, 실패 시나리오(12항) 커버리지, 시크릿/`.env` 유출 여부, 불필요한 복잡도·중복·죽은 코드.
3. 이 Task 범위 안의 문제만 고쳐줘. 범위 밖 이슈를 발견하면 코드를 고치지 말고 "새 Issue 제안"으로만 알려줘.
4. 수정 후 테스트를 재실행해서 green을 확인해줘 (같은 실패를 3라운드 넘게 고쳐도 안 되면 멈추고 보고해줘).
5. 카드 11항 DoD 체크리스트를 항목별 충족 여부 표로 보여줘.
6. 리뷰 보강 커밋을 남기고, 마무리(이슈 정리 -> push -> PR -> 머지) 진행 여부를 물어봐줘.
