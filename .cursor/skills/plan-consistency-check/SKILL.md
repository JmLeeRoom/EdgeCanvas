# 계획서 정합성 검증 (plan-consistency-check)

세 문서를 수정한 뒤 회귀 점검용으로 실행한다. 문서를 읽고 아래 항목을 표로 보고한다.

## 점검 항목

```
- [ ] 1. Task 총개수 45개, 카드(### [T-xxx])와 백로그 행 수 일치
- [ ] 2. 모든 선행 태스크 ID가 실재 (존재하지 않는 T-xxx 참조 없음)
- [ ] 3. 주차 역전 없음 (선행이 후행보다 늦은 주차 아님)
- [ ] 4. Phase↔주차 정합 (Phase A가 10~12주 배치 안 됨 등)
- [ ] 5. 공수 합계: 담당별(A/B/pair) 합 = Phase별 합 = 55.5인일
- [ ] 6. 마일스톤 게이트 Task가 해당 주차(M1=7,M2=9,M3=12) 내 완료
- [ ] 7. 헤더 번호 연속성 (중복 8.1, 누락 7.x 등 없음)
- [ ] 8. 컷 Task(T-006/T-504) 실행 공수·활성 선행에서 제외
- [ ] 9. 세 문서 간 수치 동기화 (공수·마일스톤·주차)
```

## 검증 방법

- **개수/헤더**: `단위구현계획서.md`에서 `^### \[T-` 카드 수, `^## 제N장`/`^### N.` 헤더를 grep해 연속성 확인.
- **선행·주차·공수**: 제2장 백로그 표를 파싱한다. 각 행(`| T-xxx | ... | 담당 | 공수 | 선행 | 주차 |`)에서:
  - 선행 ID가 표에 존재하는지
  - 선행의 주차 ≤ 해당 Task 주차인지 (컷/모의 로그 예외 명시)
  - 담당별 공수 합, Phase별 공수 합이 각각 55.5인지 (페어는 2인일 환산)
- **마일스톤**: 완료 주차 ≤7→M1, 8~9→M2, ≥10→M3 매핑이 `GITHUB_워크플로_가이드.md` 6.3절과 일치하는지.
